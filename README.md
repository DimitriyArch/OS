## 1. Git Overview
Git — це система керування версіями, що дозволяє зберігати історію змін коду, працювати над проектом кільком розробникам одночасно та повертатися до попередніх версій у разі помилок.

**Основні команди:**
- `git add` - підготовка файлів.
- `git commit` - збереження версії.
- `git push` - відправка у хмару.

## 2. What is a Commit?
Комміт — це логічна одиниця збереження змін. Він дозволяє бачити різницю (diff) між поточним та попереднім станом файлів, що робить процес розробки прозорим.

## 3. English Section
### Vocabulary
- **Repository** — a storage location for software packages.
- **Branch** — a parallel version of a repository.
- **Merge** — joining two or more development histories together.
- **Remote** — a shared repository (on GitHub/GitLab) used for collaboration.

### Conclusions
During this work-case, we have successfully set up a shared Git repository. We learned how to track changes using commits and how to collaborate as a team. Git is an essential tool for modern operating systems development and software engineering in general.
