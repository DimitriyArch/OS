## 1. Робота в Ubuntu та середовищі GNOME

### 1.1. Структура робочого простору
* **Основне меню (Activities):** Перегляд відкритих вікон, запуск програм, перехід між робочими столами.
* **Панель швидкого доступу (Dock):** Розташована ліворуч. Дозволяє закріплювати улюблені програми та перемикатися між активними вікнами.
* **Верхня панель (Top Bar):** Містить час, системні індикатори (мережа, звук, батарея) та меню завершення роботи.
* **Пошук:** Інтегрований у режим «Огляд», дозволяє миттєво знаходити програми та файли.
* **Робочі столи (Workspaces):** Динамічні простори для групування вікон.

### 1.2. Запуск програм
Програми запускалися наступними способами:
1. Через сітку програм (**Applications Grid**).
2. Через **Пошук** (введення назви з клавіатури).
3. Через **Dock** (клік по закріпленій іконці).
4. Через **Термінал** (введення імені команди).

### 1.3. Вихід із системи та завершення роботи
* **Зміна користувача на root:** Через меню користувача (Вийти) або в терміналі командою `sudo -i`.
* **Перезавантаження/Вимкнення:** Меню користувача -> «Вимкнути / Вийти» -> Підтвердження дії.

## Відповіді на контрольні запитання

### Наведіть приклади серверних додатків Linux для сервера баз даних, серверів розсилки повідомлень та файлообмінників.

Серед серверних додатків Linux для серверів баз даних можна виділити MySQL та MariaDB як популярні реляційні СУБД, PostgreSQL як потужну об’єктно-реляційну СУБД, MongoDB як документоорієнтовану NoSQL базу даних та Redis для швидкого кешування і зберігання даних; для серверів розсилки повідомлень використовують Postfix, Exim та Sendmail як поштові сервери, Dovecot для отримання пошти через IMAP/POP3 та RabbitMQ для обміну повідомленнями між додатками; а серед файлообмінників поширені Samba для обміну файлами між Linux та Windows, NFS для Linux/Unix систем, Nextcloud для хмарного зберігання та обміну файлами, а також FTP-сервери на кшталт vsftpd і ProFTPD.

### Порівняйте оболонки Bourne, C, Bourne Again (Bash), the tcsh, Korn shell (Ksh) та zsh.

Оболонки Linux та Unix відрізняються синтаксисом команд, можливостями скриптування та інтерактивними функціями. Bourne shell (sh) — це класична оболонка з базовим синтаксисом команд і підтримкою простого скриптування, вона є стандартом для багатьох систем, але не має розширених інтерактивних можливостей. C shell (csh) орієнтована на користувачів, знайомих із мовою програмування C, має зручний синтаксис для умов і циклів, підтримує історію команд і псевдоніми, проте її скрипти менш портативні і інколи складніші у відладці. Bourne Again shell (Bash) поєднує функції Bourne shell та додаткові можливості, такі як розширена історія команд, автодоповнення, масиви та потужне скриптування, і є найпоширенішою оболонкою Linux. Tcsh є вдосконаленою версією C shell з інтерактивними поліпшеннями, зокрема автодоповненням і редагуванням командного рядка. Korn shell (Ksh) суміщає функції Bourne shell і C shell, пропонуючи ефективне скриптування, розширені змінні та арифметику, а також інтерактивні можливості на рівні Bash.

### Для чого потрібен менеджер пакетів. Які менеджери пакетів ви знаєте у Linux?

Менеджер пакетів у Linux потрібен для спрощення встановлення, оновлення, видалення та управління програмним забезпеченням і його залежностями, тобто він автоматично завантажує потрібні пакети, перевіряє сумісність та підтримує систему в актуальному стані. Серед відомих менеджерів пакетів у Linux є APT (для Debian та Ubuntu), DPKG (також для Debian-подібних систем), YUM та DNF (для RedHat, Fedora, CentOS), RPM (для RedHat-подібних систем), Pacman (для ArchLinux), Zypper (для openSUSE) і Snap або Flatpak для універсальної установки пакетів незалежно від дистрибутива.

### Які засоби безпеки використовуються в Linux?

У Linux використовується низка засобів безпеки, що забезпечують захист системи та даних користувачів. Основним механізмом є система прав доступу до файлів і каталогів, яка визначає дозволи на читання, запис і виконання для власника, групи та інших користувачів. Для розширеного контролю доступу застосовуються ACL (Access Control Lists), що дозволяють задавати більш детальні правила доступу. Важливим засобом є автентифікація користувачів за допомогою паролів, ключів SSH та модулів PAM (Pluggable Authentication Modules), які керують процесом входу в систему. Для ізоляції та додаткового контролю процесів використовуються системи SELinux і AppArmor, що обмежують дії програм навіть у разі їх компрометації. Також у Linux застосовуються мережеві брандмауери (iptables, nftables, firewalld), засоби шифрування даних (LUKS, GnuPG), журнали подій (log-файли) для моніторингу безпеки та регулярні оновлення системи через менеджери пакетів, які закривають відомі вразливості.

### Чому використання віртуалізації зараз стало таким актуальним?

Використання віртуалізації стало дуже актуальним через кілька ключових причин. По-перше, вона дозволяє ефективніше використовувати апаратні ресурси, запускаючи кілька віртуальних машин на одному фізичному сервері, що знижує витрати на обладнання. По-друге, віртуалізація забезпечує ізоляцію середовищ, завдяки чому різні додатки або системи можуть працювати незалежно, без конфліктів між ними. По-третє, вона спрощує тестування, розгортання і масштабування програмного забезпечення, бо можна швидко створювати та видаляти віртуальні машини чи контейнери. Додатково віртуалізація підвищує безпеку та відмовостійкість, оскільки системи можна легко резервно копіювати, переносити або відновлювати у разі збоїв. Нарешті, сучасні тренди, такі як хмарні сервіси та DevOps, значною мірою базуються на віртуалізації, що робить її практично незамінною у сучасній ІТ-інфраструктурі.

### Як ви розумієте поняття контейнеризації?

Контейнеризація — це технологія, яка дозволяє запускати додатки у ізольованих середовищах, званих контейнерами, на одній операційній системі. На відміну від повноцінної віртуальної машини, контейнер не містить окремої ОС, а використовує ядро хост-системи, що робить його легким і швидким у запуску. Кожен контейнер містить лише потрібні для роботи додатку бібліотеки, залежності та конфігурації, що забезпечує передбачувану поведінку програм незалежно від того, на якій системі він запущений. Контейнеризація широко застосовується для швидкого розгортання, масштабування і переносу додатків, а також для ізоляції процесів і забезпечення безпеки. Найпопулярніші інструменти контейнеризації — Docker та Podman, а в масштабованих кластерах часто використовується Kubernetes.

### Які переваги/недоліки використання програмного забезпечення з відкритим кодом?

Використання програмного забезпечення з відкритим кодом має низку переваг і недоліків. Серед основних переваг варто виділити безкоштовність або низьку вартість ліцензії, що дозволяє економити кошти на придбанні програм; гнучкість та кастомізацію, оскільки код можна змінювати під власні потреби; прозорість і безпеку, бо відкритий код можна перевірити на наявність вразливостей; а також широку спільноту підтримки, яка допомагає вирішувати проблеми та розвивати програму. До недоліків відносять потребу в технічних знаннях для налаштування та підтримки, неоднорідну або неповну документацію у деяких проєктах, а також можливу нестабільність або обмежену підтримку, особливо якщо проєкт маловідомий або припинив активний розвиток.

### Скільки активних віртуальних консолей (терміналів) може бути у процесі роботи Linux по замовчуванню. Як їх викликати та між ними перемикатися? Наведіть приклади?

У Linux за замовчуванням доступно шість активних текстових віртуальних консолей, які викликають і між якими перемикаються за допомогою комбінацій клавіш `Ctrl + Alt + F1…F6`, а графічний сеанс зазвичай знаходиться на `F7`. Наприклад, `Ctrl + Alt + F2` дозволяє перейти на другу консоль, виконати команди під іншим користувачем і повернутися у графічний режим через `Ctrl + Alt + F7`.

### Яка віртуальна консоль (термінал) виконує функцію графічної оболонки?

У Linux за замовчуванням функцію графічної оболонки виконує віртуальна консоль `F1`/`F7`/`F8`, куди можна перейти комбінацією `Ctrl + Alt + F7` для роботи в графічному середовищі.

### Чи можлива реєстрація в системі Linux декілька разів під одним і тим же системним ім’ям? Які переваги це може надати?

У Linux можлива одночасна реєстрація кількох сеансів під одним і тим же системним ім’ям користувача, наприклад через різні віртуальні консолі або графічні сесії. Це дозволяє паралельно виконувати різні завдання, розділяти роботу в окремих сеансах, тестувати команди або програми без переривання основної роботи, а також спрощує віддалений доступ до системи під тим самим користувачем з кількох терміналів або через SSH.
